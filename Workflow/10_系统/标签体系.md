# 标签体系

> 核心原则：**标签不是分类，是"我接下来想怎么找到它"**

---

## 一、标签设计原则

1. **少而精准**：不要一个文件打 10 个标签
2. **标签 ≠ 文件夹**：标签解决"跨文件夹查找"的问题
3. **三类标签**：事务类型 / 状态 / 紧急程度

---

## 二、事务类型标签

用于标记"这是什么类型的事"：

| 标签 | 用途 |
|------|------|
| `#报销` | 所有报销相关 |
| `#盖章` | 学院 / 学校盖章 |
| `#项目申请` | 项目申报 |
| `#差旅` | 出差、外出 |
| `#会议` | 学术会议 |
| `#采购` | 设备 / 材料采购 |
| `#合同` | 合同签订 |
| `#发票` | 发票相关问题 |

---

## 三、状态标签

用于标记"这件事现在到什么阶段了"：

| 标签 | 用途 |
|------|------|
| `#待办` | 还没开始 |
| `#进行中` | 正在准备材料 |
| `#已提交` | 已提交，等结果 |
| `#被退回` | 被打回来了，需修改 ⚠️ |
| `#已完成` | 全部搞定 ✓ |

---

## 四、角色标签

用于标记"这事要找谁"：

| 标签 | 用途 |
|------|------|
| `#导师` | 需要导师签字 / 审核 |
| `#学院` | 需要学院办公室处理 |
| `#财务处` | 需要财务处处理 |
| `#科研秘书` | 需要科研秘书协助 |

---

## 五、紧急标签（建议）

偶尔会用：

| 标签 | 用途 |
|------|------|
| `#紧急` | 有截止时间，不能拖 |
| `#踩坑` | 这次遇到问题了，值得复盘 |

---

## 六、什么文件打什么标签

### 流程类文件（01-05 文件夹）

只用类型标签：

```yaml
---
tags:
  - 报销
  - 差旅
---
```

### 事务记录（08_事务记录）

必须打类型 + 状态：

```yaml
---
tags:
  - 报销
  - 差旅
  - 进行中
---
```

### 联系人

用角色标签：

```yaml
---
tags:
  - 财务处
  - 联系人
---
```

### 问题 / 复盘

标记踩坑：

```yaml
---
tags:
  - 报销
  - 踩坑
  - 被退回
---
```

---

## 七、标签使用频率建议

- 每个文件 **1-3 个标签**，不要多
- 状态标签要随时更新（`#进行中` → `#已完成`）
- 每月看一次 `#被退回` 和 `#踩坑`，总结高频问题
