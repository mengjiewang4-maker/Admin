# 横向专用项目 · 全流程

> 北京理工大学 · 横向科研项目（专用类）业务流程
> 系统：专用项目管理系统
> 主管：逸夫楼 101 董怡（需带合同 + 备案表）

---

## 1. 适用场景

- 企业/单位委托的横向科研项目（含涉密项目）
- 包括 SM 项目、内部项目
- 走专用项目管理系统

---

## 2. 流程总览

```
①申报 → ②立项 → ③过程管理 → ④外协业务 → ⑤结题
```

---

## 3. 阶段一：申报阶段

### 流程

```
系统填报 → 学院审核(定密+材料) → 科研院审核 → 用印编号 → 教师服务大厅盖章
```

### 系统填报

专用项目管理系统 → 新增项目 → 勾选「申报阶段」+「横向」

- 填写项目基本信息、用印事项及材料明细、用印类型及数量
- 提交后生成业务追踪号
- 打印进款审批表，补充填写项目名称、密级

### 学院审核

- 携带：进款审批表 + 申报材料
- 线下至学院完成定密 + 材料审核
- 学院线上审批，编制申报号 → 流转至科研院

### 科研院审核 + 盖章

- 携带：进款审批表 + 申报材料 → 科研院复核
- 科研院线上审批，生成用印编号
- 凭用印编号 → 教师服务大厅盖章

### 🔁 再次用印

"申报阶段再次用印"端口 → 学院审 → 科研院审 → 用印编号 → 盖章

---

## 4. 阶段二：立项业务

### 流程

```
系统填报 → 学院审核(编制档案号) → 科研院审核 → 用印编号 → 盖章 → 归档
```

### 系统填报

专用系统 → 新增项目 → 勾选「正式立项」

- 「是否有档案号」选「否」→ 由学院后续编制
- 填写基本信息、用印事项
- 打印进款审批表，补充合同编号、密级

> 📌 前期已线下完成审批的：「是否有档案号」选「是」，仅系统备案。

### 学院审核

- 携带：进款审批表 + 合同 + 保密登记备案表（SM 项目）
- 学院完成定密 + 材料审核
- 编制档案号

### 归档要求

| 项目类型 | 归档材料 |
|----------|----------|
| SM 项目 | 合同原件 1 份 + 进款审批表 + 保密登记备案表 + 电子版 SM 项目信息登记表 + 信息交换单 2 份 + 载体交接单（如合同为 SM 载体） |
| 内部项目 | 合同原件 1 份 + 进款审批表 |

---

## 5. 阶段三：立项后过程管理

`正式立项过程管理` 端口 → 填写用印事项 → 学院审核 →（含财务报告则加计财部）→ 科研院审核 → 盖章

与 GF 纵向项目过程管理逻辑一致。

---

## 6. 阶段四：外协业务

### 前置条件 ⚠️

主合同的「盖章」「归档」状态标记**均必须为「是」**。

### 流程

```
出款承诺 → 系统填报 → 学院审核(定密) → 科研院审核 → 用印编号 → 盖章 → 归档
```

### 归档要求

| 出款类型 | 归档材料 |
|----------|----------|
| SM 密出款 | 出款合同 + 出款审批表 + 保密协议书 + 协作配套单位保密资质证书复印件 + 保密监督检查表 |
| SM 非密出款 | 出款合同 + 出款审批表 |
| 内部项目 | 出款合同 + 出款审批表 |

---

## 7. 阶段五：结题业务

`专用项目结项申请` 端口 → 学院审核 →（含财务验收报告则加计财部）→ 科研院审核 → 盖章

---

---

## 9. 常见错误 ⚠️

| # | 错误 | 预防 |
|---|------|------|
| 1 | 外协前没确认主合同已归档 | 外协前在系统确认「盖章」「归档」状态均为是 |
| 2 | SM 项目用印时缺保密登记备案表 | SM 项目始终需要保密登记备案表 |
| 3 | 归档材料不齐 | 对照第 4 节归档清单逐项核对 |
| 4 | 找错主管 | 横向项目主管是 101 董怡，不是其他办公室 |
