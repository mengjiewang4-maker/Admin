# 民口纵向专用项目 · 全流程

> 北京理工大学 · 民口纵向科研项目（专用类）业务流程
> 系统：专用项目管理系统
> 包括：国家重大专项、国家重点研发计划、部委科技计划、各省市科技计划中的专用项目

---

## 1. 适用场景

- 民口纵向科研项目中的专用（涉密/内部）项目
- 由科研院基础科学研究院负责管理的项目中的专用类
- 走专用项目管理系统

---

## 2. 流程总览

```
①申报 → ②正式立项 → ③过程管理(中期) → ④出款合同 → ⑤结题验收
```

与 GF 纵向项目流程结构类似，但**申报和立项阶段不走计财部**。

---

## 3. 五个阶段速查

| 阶段 | 入口 | 审批链 | 关键判断 |
|------|------|--------|----------|
| 申报 | 新增项目→申报阶段 | 学院→科研院 | — |
| 立项 | 新增项目→正式立项 | 学院→科研院 | 前期是否线下已审批？ |
| 过程管理 | 正式立项过程管理→中期 | 学院→(计财部)→科研院 | 是否含财务报告？ |
| 出款 | 出款新增→出款承诺 | 学院→科研院 | — |
| 结题 | 结项申请 | 学院→(计财部)→科研院 | 是否含财务验收报告？ |

---

## 4. 申报阶段

### 流程
```
系统填报 → 学院(定密+审核+编制申报号) → 科研院审核 → 用印编号 → 教师服务大厅盖章
```

### 关键步骤

1. 意向申报民口纵向指南 → 报学院备案 → 专用系统提交
2. 系统：「新增项目」→ 勾选「申报阶段」→ 选择正确项目类型
3. 填写基本信息、用印事项 → 生成业务追踪号 → 打印进款审批表
4. 线下学院定密审核 → 学院编制申报号
5. 科研院复核 → 用印编号 → 教师服务大厅盖章

🔁 再次用印：「申报阶段再次用印」端口 → 学院→科研院→盖章

---

## 5. 正式立项阶段

### 流程
```
系统填报 → 学院(定密+编制档案号) → 科研院审核 → 用印编号 → 盖章
```

### 关键步骤

1. 项目通过评审 → 拟制正式合同
2. 「新增项目」→ 勾选「正式立项」→「是否有档案号」选「否」
3. 填写基本信息 → 打印进款审批表 → 补充合同编号、密级
4. 携带：进款审批表 + 合同/任务书/协议书 + 保密登记备案表 → 学院定密审核
5. 学院编制档案号 → 科研院审核 → 用印编号 → 盖章

> 📌 前期已线下办理立项审批的：「是否有档案号」选「是」，填写已有档案号，仅备案。

---

## 6. 过程管理（中期）

`正式立项过程管理` → 选择「中期」→ 学院审核 →（含财务支出报告则加计财部）→ 科研院审核 → 盖章

---

## 7. 出款合同审批

### 前置条件
项目任务书/协议书**已归档**。

### 流程
```
出款新增 → 出款承诺 → 填写合同信息 → 打印出款审批表 → 学院(定密+审核) → 科研院审核 → 用印编号 → 盖章
```

携带材料：出款审批表 + 出款合同 + 保密登记备案表 + 乙方单位资质

---

## 8. 结题验收

`结项申请` 端口 → 学院审核 →（含财务验收材料则加计财部）→ 科研院审核 → 盖章

---

## 9. 需要找谁

| 环节 | 找谁 | 备注 |
|------|------|------|
| 学院审核 | 学院科研秘书 / 定密责任人 | 线下+线上 |
| 财务审核 | 计财部 | 仅过程管理/结题含财务材料时 |
| 科研院审核 | 民口纵向项目主管 | |
| 盖章 | 教师服务大厅 / 科研院 | 凭用印编号 |

---

## 10. 常见错误 ⚠️

| # | 错误 | 预防 |
|---|------|------|
| 1 | 忘了报学院备案就去系统填报 | 意向申报后先报学院备案 |
| 2 | 项目类型选错 | 确认主管单位分类后再选 |
| 3 | 出款前没确认主项目已归档 | ⚠️ 归档是出款前置条件 |
| 4 | 忘带保密登记备案表 | 专用项目全程需要 |
